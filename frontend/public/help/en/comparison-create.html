Create Comparison

The create comparison function allows you to select two dictionary versions for detailed analysis. The system will automatically identify and count various changes, providing strong support for dictionary research and version management.

Creation Steps

1

Enter Comparison Page

Click "Version Comparison" in the navigation bar to enter the version comparison management page.

Version comparison management page

2

Click New Comparison

On the comparison list page, click the "New Comparison" button to start creating a new comparison task.

Click the New Comparison button

3

Select Version A

In the popup dialog, select the first dictionary version to compare (Version A).

Required
Version A - Select the version as baseline, usually the earlier version

Select Version A dropdown menu

4

Select Version B

Select the second dictionary version to compare (Version B), usually the newer version.

Required
Version B - Select the target version to compare

Select Version B dropdown menu

5

Start Comparison

After confirming the selected versions are correct, click the "Start Comparison" button to launch comparison analysis.

Click the Start Comparison button

6

Wait for Processing

The system will automatically process the comparison task. You can view processing progress in the comparison list.

Comparison task processing progress

Version Selection Suggestions

⭐

Time Order

It is recommended to use the earlier version as Version A and the newer version as Version B to facilitate understanding the direction of changes.

⭐

Version Differences

Select versions with significant differences for comparison to better demonstrate the analysis function.

⭐

Data Completeness

Ensure selected versions are fully parsed to avoid incomplete data affecting comparison results.

⭐

Comparison Purpose

Select appropriate versions based on research purpose. For studying changes in a specific period, you can select adjacent versions.

Comparison Parameters

Required
Version A - The baseline dictionary version, usually the earlier version or reference version
Required
Version B - The target dictionary version for comparison, usually the newer version
Optional
Comparison Name - Set a descriptive name for the comparison task for easy identification
Optional
Notes - Add purpose and background information for the comparison

Waiting

🔄

Processing

✅

Completed

❌

Notes

Version Selection

  • Ensure selected versions belong to the same dictionary
  • Avoid selecting completely identical versions
  • Check if versions have completed parsing
  • Recommended to select versions with actual differences

Processing Time

  • Large dictionary comparisons may take longer
  • You can close the page, system will continue processing
  • Will be displayed in comparison list when processing is complete
  • Can create multiple comparison tasks simultaneously

Result Viewing

  • View results immediately after processing is complete
  • Results are automatically saved and can be viewed at any time
  • Can quickly filter by status and statistics
  • Can rerun comparison analysis

FAQ

Can I compare versions from different dictionaries?
Currently does not support cross-dictionary comparison, only different versions of the same dictionary can be compared.
What to do if comparison fails?
Check version status to ensure both versions are fully parsed. You can recreate the comparison task.
Can I create multiple comparisons at the same time?
Yes, the system supports processing multiple comparison tasks simultaneously, but this may affect processing speed.
How long are comparison results saved?
Comparison results are saved long-term. You can return to view statistics and difference content at any time.
