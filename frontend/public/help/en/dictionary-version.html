Best Practices

⭐

Version Naming

Use clear version naming conventions, such as "7th Edition", "2023 Edition", etc., for easy identification and management.

⭐

Regular Backup

Important versions are recommended to be backed up to avoid data loss from accidental deletion.

⭐

Timely Comparison

Compare with the previous version promptly after a new version is completed to discover and correct issues.

⭐

Record Changes

Record main changes in the version description for subsequent tracking.

FAQ

Can multiple versions be uploaded at the same time?
Yes, but it is recommended to wait for one version to be fully processed before uploading the next version to avoid high system load.
What to do if parsing fails?
Check if the file format is correct and if the parsing configuration is appropriate. You can view error logs to understand specific issues.
Is there a limit on the number of versions?
Different limits apply based on user plan. Regular users can have up to 10 versions, premium users have more.
