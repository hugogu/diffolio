Taxonomy Dictionary Overview

The taxonomy dictionary function supports users in uploading custom classification systems, establishing mapping relationships between vocabulary and categories, for use in semantic analysis and thematic retrieval in dictionary research. The platform does not have built-in fixed classifications; users define and upload them according to their research needs.

Classification System Structure

The classification system adopts a hierarchical structure, supporting multi-level categories:

L1

Level 1 Category

Highest level categories, such as "Technology", "Culture"

  • Each Level 1 category contains multiple Level 2 categories
  • Defines basic scope of classification
  • Provides overall framework for classification
L2

Level 2 Category

Intermediate level categories, such as "Computer Technology", "Literature and Art"

  • Refines content of Level 1 categories
  • Connects specific vocabulary and high-level categories
  • Provides more precise classification positioning
L3

Level 3 Category

Specific categories, such as "Programming Languages", "Classical Literature"

  • Finest granularity classification level
  • Directly associated with specific vocabulary
  • Supports precise semantic positioning

Data Formats

CSV

CSV Format

Comma-separated values format, suitable for batch data processing

  • Vocabulary,Category1,Category2,Category3
  • Supports direct editing in Excel
  • Easy batch import
  • Simple and intuitive format
JSON

JSON Format

Structured data format, suitable for program processing

  • Hierarchical data structure
  • Supports complex mapping relationships
  • Easy API integration
  • Supports metadata
XML

XML Format

Markup language format, suitable for complex classification systems

FAQ

How to create a custom classification system?
You can create a new classification system on the classification management page, defining classification structure and mapping relationships.
How many levels of classification hierarchy are supported?
The system supports multi-level classification. It is recommended not to exceed 5 levels to avoid being too complex and affecting usability.
Can classification data be exported?
The current version supports taxonomy upload. Export functionality is planned for future versions.
