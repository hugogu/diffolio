Configuration Management Overview

Configuration management allows you to create and manage dictionary parsing rules. Through custom configurations, you can improve the accuracy and efficiency of dictionary file parsing. The system provides flexible configuration templates and a powerful rule editor.

Configuration Management Interface

Configuration management list interface

The configuration management interface displays all available parsing configurations, including system built-in configurations and user custom configurations.

Configuration Rule Types

Required
Entry Recognition Rule - Defines how to recognize and extract entry names, usually using regular expressions
Required
Definition Extraction Rule - Defines how to extract definition content, supports multi-line matching
Optional
POS Recognition Rule - Defines extraction patterns for part-of-speech annotations
Optional
Example Recognition Rule - Defines start and end markers for examples
Optional
Pinyin Recognition Rule - Defines extraction method for pinyin information
Optional
Extended Field Rule - Defines extraction rules for custom fields

Configuration Creation Workflow

1

2

3

After confirming the configuration is correct, save it. It can then be used during dictionary upload.

FAQ

Not familiar with regular expressions, what to do?
The system provides rich templates and examples. You can start with simple rules and gradually learn regular expression syntax.
Can configurations be shared with other users?
Currently configurations are managed under the current user account only. To share, other users can create similar configurations based on the same templates.
How to improve parsing accuracy?
Through continuous testing and optimization of rules, using more sample data for validation, and considering edge case handling.
